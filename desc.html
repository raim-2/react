리액트 사용 이유 : 사용자 인터페이스 구현을 위한 자바스크립트 라이브러리 → 재로딩없이 업데트해줌
사용자의 사용 경험을 증가, 부드러운 화면 전환에 도움
자바스크립트로만 작업하는 것은 번거롭고 오류가 더 많다.더 쉽게 인터페이스를 구축해주기 때문에 리액트를 사용함

app.js에 핵심코드가 들어가 있는데, html과 자바스크립트 코드를 섞어 쓰는 것이 특징이다.
리액트 코드는 선언형으로 작성해야됨. 과정은 x [작동원리/ 자바스크립트는 명령형 - 거쳐야 할 단계를 알려줌]
즉 조건과 목표 상태, 상태 변경 조건만 정의하면 리액트가 알아서 처리해줌

1. 리액트 생성방법은 codesandbox를 사용할경우 react.new
2. Vite를 통한 생성 방법 / create react app을 통한 생성 방법이 있음
리액트에서 사용하는 문법은 jsx → 브라우저에서 사용가능한 코드로 변환(이걸 vite와 같은 도구가 도와줌)

컴포넌트 - 재사용이 가능한 요소 / UI의 한 부분이 된다
리액트는 컴포넌트를 결합해 만드는 것 - 관리하기 쉽게 작게 분리한 것
비슷한 코드가 겹쳐있어 - 개발이 용이하다.

npm install을 해서 다운을 받고, npm run dev를 입력해 로컬 서버를 실행시킬 수 있다. 종료하려면 ctrl+c를 입력하면 됨 jsx라는 자바스크립트 문법 확장자를 사용함 → 자바스크립트 코드 안에 html을 입력함 브라우저에서는 사용이 불가하다. 그래서 브라우저 도달 전 서버에서 코드가 변환되어 전달됨 리액트는 모든 jsx 코드를 분석 결합하여 전반적인 DOM을 생성한다.
